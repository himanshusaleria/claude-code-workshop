## Key Points to Cover

### 1. The current pain (hook)
- Average designer uses 3-4 tools per project
- Time wasted on file exports, format conversion, manual BOQ
- Real stat: 30% of project time is spent on admin, not design

### 2. Why this is getting worse
- Teams are growing, collaboration is harder
- Clients expect 3D walkthroughs and instant revisions
- Remote/hybrid work makes file-sharing even more painful

### 3. What cloud-native design looks like
- One tool for 2D, 3D, collaboration, and BOQ
- Work from anywhere, no heavy software installs
- Real-time collaboration (the "Google Docs for design" analogy)
- Auto-generated BOQs that update when the design changes

### 4. Case study / example
- Use BuildCraft Interiors as an example (with permission)
- Before: 4-5 revision cycles, 3 days per BOQ, lost a client due to slow turnaround
- After: Same-day revisions, instant BOQ, 3D client walkthroughs
- Result: 40% faster project delivery, 2 more projects per quarter

### 5. What to look for in a design tool (buying guide angle)
- 2D + 3D in one workflow
- Real-time collaboration
- Auto BOQ generation
- Client-facing features (3D walkthrough, comments)
- File import (DWG, SKP support)
- Indian pricing / local support
